JavaScript version ES6: JS files vs JS modules


a_JS_file.js and b_JS_file.js are both JavaScript files

c_JS_module.js and b_JS_module.js are also both JavaScript files, even though they are named ?_JS_module.js there is NO browser directive to load them as modules in the <script> tags

Open the console to see the output. You will see that:

  • a_JS_files.js function a_func_1 calls b_JS_file.js function b_func_1
  • c_JS_module.js function c_func_1 calls d_JS_module.js function d_func_1

Now direct the browser that c_JS_module.js and b_JS_module.js are both JavaScript modules by adding type="module" to their <script> tags. IE:

  • <script type="module" src="c_JS_module.js"></script>
  • <script type="module" src="d_JS_module.js"></script>

Open the console to see the output. You will see that that the browser failed to locate the c_JS_module.js function c_func_1

This is because with ES6 JS modules all constants variables and functions are by default local to the module they are defined in.
